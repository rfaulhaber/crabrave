  - Support for videos (mp4, webm, mov, etc.)
  - Support for audio (mp3, ogg, wav, flac, etc.)

- Multipart/form-data implementation:
  - ~post_multipart()~ for creating posts with media
  - ~put_multipart()~ for editing posts with media
  - Follows Tumblr API format: "json" field + media fields by identifier
- Builder methods:
  - ~.media_source(identifier, source)~ - explicit identifier control
  - ~.add_image(source)~ - convenience method with auto-generated IDs
  - ~.add_video(source)~ - convenience method for videos
  - Automatic JSON vs multipart selection based on media presence
- Test coverage: 8 new tests for media upload functionality
* API Refactoring [1/1]
** DONE Post Operations Moved to Blog Handler
All post operations now go through ~crab.blogs("blog-name")~ for consistency:
- Removed ~crab.posts()~ accessor from main client
- Removed ~Posts~ struct (was redundant with blog parameter)
- Added ~crab.blogs("foo").post("id")~ - returns BlogPost handler
  - ~.get()~ - fetch the post
  - ~.delete()~ - delete the post
  - ~.edit()~ - returns EditPostBuilder
  - ~.mute()~ - mute notifications
- Added ~crab.blogs("foo").create_post()~ - returns CreatePostBuilder
- Added ~crab.blogs("foo").reblog(id, key)~ - returns ReblogBuilder

Benefits:
- Consistent API: all blog operations start with ~crab.blogs()~
- Blog specified once instead of repeated in each operation
- More natural API: ~crab.blogs("foo").post("123").get()~ vs ~crab.posts().get("foo", "123")~
